Grade your MCP server in one command.

Models pick the wrong tools, token bills balloon, and servers break their own protocol — none of it visible from your server's code. Jig makes it visible, measurable and regression-safe, then writes you a report card you can send to someone.

Why a grade?

An MCP server "works" only if a model understands its tools, picks the right one and fills the arguments. That surface is probabilistic — and the ecosystem's numbers show how often it goes wrong.

What it does

One session, many questions answered. Every number is exact or explicitly labelled as an approximation — never silently wrong.

jig check

Is my server well-built? One graded verdict over five weighted dimensions, ranked fixes, and a self-contained HTML report card written by default.

jig context

What does the model actually see? The exact tool-use request body, token-annotated, per provider dialect. No API key needed — nothing is sent anywhere.

jig budget

What does the server cost in context, per tool, per model tokenizer — graded against a real ecosystem distribution.

jig bench

Which tool does a real model pick for a task? N-run selection distributions with argument validation. Bring your own key.

jig eval

.jig regression suites in git: prompt → expected tool call, selection rates never single-run booleans, CI gates.

jig auth

OAuth conformance for HTTP servers — RFC-cited checks of metadata, challenge, PKCE and DCR. No login flow, no credentials.

The standards behind the grade

Every rule jig enforces is published, evidence-cited, and paired with the exact command that verifies it — including honest labels where nothing can verify it yet.

The rubric

rubric-v1 — weights and penalty tables in the open. Arguable on purpose; tell us where it's wrong.

What this doesn't do (yet)

The desktop workbench is in design — an interactive prototype exists; the CLI is what ships today. `jig auth` doesn't log in — it grades the discoverable OAuth surface; the authorization-code + PKCE flow is on the roadmap. Description quality is deterministic heuristics, never an LLM verdict, and it says so in every report. The ecosystem leaderboard is being generated and will land at /leaderboard/. We publish corrections as loudly as findings — see the census.
